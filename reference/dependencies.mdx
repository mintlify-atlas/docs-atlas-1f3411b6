## Locked Versions

All plugins are locked to specific commits in `lazy-lock.json`. This ensures:

- **Reproducibility**: Same versions across all installations
- **Stability**: Prevents breaking changes from automatic updates
- **Control**: Updates only when explicitly run with `:Lazy update`

### Updating Plugins

To update all plugins:

```vim
:Lazy update
```

To update a specific plugin:

```vim
:Lazy update plugin-name
```

After updating, test thoroughly before committing the updated `lazy-lock.json`.

## Plugin Loading Strategy

Plugins are loaded using various strategies for optimal startup time:

- **Lazy-loaded on event**: Load when specific Neovim events occur
- **Lazy-loaded on keymap**: Load when specific keys are pressed
- **Lazy-loaded on command**: Load when a command is run
- **Lazy-loaded on filetype**: Load for specific file types
- **Eager-loaded**: Load immediately at startup (only for essential plugins)

## Adding New Plugins

To add a new plugin:

1. Create a new file in `lua/leyland/plugins/` or add to an existing file
2. Return a table with the plugin specification:
   ```lua
   return {
     "author/plugin-name",
     config = function()
       -- Setup code
     end,
   }
   ```
3. Restart Neovim - lazy.nvim will auto-install the plugin
4. The plugin will be added to `lazy-lock.json` automatically

## Removing Plugins

To remove a plugin:

1. Delete or comment out the plugin specification
2. Run `:Lazy clean` to remove the plugin files
3. Restart Neovim

## Dependencies Graph

Some plugins depend on others:

- **Telescope** → plenary.nvim, telescope-fzf-native.nvim, nvim-web-devicons
- **Completion** → nvim-cmp + multiple cmp sources + LuaSnip + friendly-snippets + lspkind
- **LSP** → nvim-lspconfig + mason.nvim + mason-lspconfig + mason-tool-installer
- **File Explorer** → nvim-tree.lua + nvim-web-devicons + nvim-lsp-file-operations
- **Git** → neogit + diffview.nvim (both depend on plenary.nvim)

Dependencies are handled automatically by lazy.nvim when specified in plugin configs.
