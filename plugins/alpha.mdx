Alpha is a fast and customizable startup screen for Neovim. nvim-luffy includes a custom One Piece-themed ASCII art dashboard that greets you when you start Neovim.

## Features

- Custom Luffy ASCII art header
- Quick action buttons for common tasks
- Inspirational One Piece quote
- Loads on Neovim startup
- No folding interference

## Dashboard Preview

When you start Neovim without opening a file, you'll see:

```
[Luffy ASCII Art - A detailed character portrait]

"Maybe the One Piece was the commits you made along the way."

  Find file
  Recent files
  Quit
```

## Configuration

The Alpha dashboard configuration is located at `lua/leyland/plugins/alpha.lua`:

```lua lua/leyland/plugins/alpha.lua
return {
  "goolord/alpha-nvim",
  event = "VimEnter",
  config = function()
    local alpha = require("alpha")
    local dashboard = require("alpha.themes.dashboard")

    dashboard.section.header.val = {
      -- ASCII art lines (Luffy character)
      -- [42 lines of ASCII art]
    }

    -- Set menu buttons
    dashboard.section.buttons.val = {
      dashboard.button("f", "  Find file", ":Telescope find_files <CR>"),
      dashboard.button("r", "  Recent files", ":Telescope oldfiles <CR>"),
      dashboard.button("q", "  Quit", ":qa<CR>"),
    }

    dashboard.section.footer.val = {
      "",
      '"Maybe the One Piece was the commits you made along the way."',
      "",
    }

    dashboard.config.layout = {
      { type = "padding", val = 2 },
      dashboard.section.header,
      { type = "padding", val = 2 },
      dashboard.section.footer,
      { type = "padding", val = 2 },
      dashboard.section.buttons,
      { type = "padding", val = 1 },
    }

    dashboard.section.footer.opts.hl = "Comment"
    alpha.setup(dashboard.opts)

    -- Disable folding on alpha buffer
    vim.cmd([[autocmd FileType alpha setlocal nofoldenable]])
  end,
}
```

## Dashboard Buttons

The dashboard includes three quick action buttons:

| Key | Action | Command |
|-----|--------|--------|
| `f` | Find file | Opens Telescope file finder |
| `r` | Recent files | Opens Telescope recent files |
| `q` | Quit | Exits Neovim |

Simply press the key to trigger the action.

## The ASCII Art

The default ASCII art is a detailed portrait of Luffy from One Piece, created using Braille characters for smooth curves and details. The art is 42 lines tall and carefully crafted to look good in any terminal.

## Advanced Features

### Dynamic Content

Make the dashboard show dynamic information:

```lua
-- Show current date/time
local datetime = os.date("%Y-%m-%d %H:%M")
dashboard.section.footer.val = {
  "",
  '"Maybe the One Piece was the commits you made along the way."',
  "",
  datetime,
}

-- Show git branch
local branch = vim.fn.system("git branch --show-current 2>/dev/null | tr -d '\n'")
if branch ~= "" then
  table.insert(dashboard.section.footer.val, "Branch: " .. branch)
end
```

### Session Management

Add session restore button:

```lua
dashboard.button("s", "  Restore session", ":lua require('persistence').load()<CR>")
```

### Plugin Statistics

Show plugin count:

```lua
local lazy_stats = require("lazy").stats()
local plugins_text = "  " .. lazy_stats.count .. " plugins"

table.insert(dashboard.section.footer.val, plugins_text)
```

## Disabling the Dashboard

If you prefer not to see the dashboard:

### Option 1: Disable the Plugin

Comment out or remove the alpha plugin in your config:

```lua
-- return {
--   "goolord/alpha-nvim",
--   ...
-- }
```

### Option 2: Skip Dashboard

Always open Neovim with a file:

```bash
nvim .          # Open current directory
nvim file.lua   # Open specific file
```

## Autocmd Details

```lua
vim.cmd([[autocmd FileType alpha setlocal nofoldenable]])
```

This prevents folding from being enabled on the dashboard, ensuring the ASCII art displays correctly.

## Tips and Tricks

1. **Quick start**: Press `f` immediately after opening to find files
2. **Muscle memory**: The button keys (`f`, `r`, `q`) match common commands
3. **No file open**: The dashboard only shows when no file is open
4. **Theme aware**: The dashboard respects your colorscheme
5. **Fast loading**: Loads on `VimEnter` event, doesn't slow down startup

## Troubleshooting

### ASCII Art Looks Broken

- Ensure your terminal supports UTF-8 and Braille characters
- Try a different terminal or font
- Reduce terminal font size if art is too large

### Dashboard Not Showing

- Check if you're opening Neovim with a file: `nvim file.lua`
- Ensure the plugin is installed: `:Lazy`
- Check for errors: `:messages`

### Buttons Not Working

- Ensure dependencies (Telescope) are installed
- Check for errors in the command strings
- Try pressing the key in normal mode

### Wrong Colors

- Check your colorscheme supports the highlight groups
- Try setting custom highlight groups (see Color Customization tab)
- Some themes may not define `Comment` highlight
